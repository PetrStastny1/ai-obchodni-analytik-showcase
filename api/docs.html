API dokumentace – AI analytika pro BI dashboard

Tato služba je “AI vrstva” nad databází sales. Umožňuje zadat dotaz v češtině (nebo angličtině), převést ho na SQL, dotaz provést nad MySQL a vrátit nejen data, ale i srozumitelné shrnutí (insighty + doporučení). Typické použití je v obchodní a marketingové analytice: tržby, top produkty, období, trendy a rychlé ad-hoc dotazy bez nutnosti psát SQL ručně.

Jak to funguje
  • 1) Dotaz – napíšeš otázku v přirozeném jazyce (např. „Jaké jsou tržby za posledních 30 dní?“).
  • 2) SQL – služba vygeneruje odpovídající SQL (filtry, agregace, group-by).
  • 3) Data – SQL se provede nad MySQL a vrátí se výsledná tabulka (JSON).
  • 4) Shrnutí – AI doplní interpretaci: hlavní zjištění, kontext a doporučení, co řešit dál.
Příklady dotazů
  • „Kterých 5 produktů udělalo nejvyšší tržby za poslední kvartál?“
  • „Porovnej tržby Macallan vs. Don Papa za poslední měsíc.“
  • „Které dny v týdnu prodáváme nejvíc?“
  • „Najdi produkty s propadem tržeb oproti minulému měsíci.“
Pozn.: Služba vrací také vygenerované SQL pro transparentnost a kontrolu výstupů.
Endpointy
GET /api/health
Health-check služby (monitoring / ověření, že služba běží).
POST /api/analyze
Hlavní analytický endpoint. Dotaz v češtině → SQL → MySQL → výsledek + AI shrnutí.
Request body – /api/analyze
  • question – dotaz v přirozeném jazyce (povinné)
  • limit – volitelný limit velikosti výsledku (default 100)
Response body
  • sql – vygenerovaný SQL dotaz (transparentnost / debug)
  • result – výsledná data (JSON pole objektů)
  • summary – shrnutí a doporučení v češtině
  • chart – volitelně návrh grafu (labels/values), pokud je to relevantní
Příklad requestu
{
  "question": "Jaké jsou tržby za posledních 30 dní a top 5 produktů?",
  "limit": 100
}
Příklad response
{
  "sql": "SELECT ...",
  "result": [{ "...": "..." }],
  "summary": "Shrnutí v češtině...",
  "chart": {
    "type": "bar",
    "metric": "revenue",
    "labels": ["..."],
    "values": [123, 456],
    "unit": "CZK"
  }
}
